# Локальная разработка — вход через Telegram

Если на localhost показывается ошибка «Настройте TELEGRAM_LOGIN_BOT», проверьте:

## 1. Запуск проекта

Запускайте **именно pifagor-trade-website** (а не pifagor-trade-redesign):

```bash
cd pifagor-trade-website
npm start
```

Откройте сайт в браузере по адресу **http://localhost:3000** (или порт из вывода терминала).

## 2. Файл .env

В папке `pifagor-trade-website` должен быть файл `.env` с переменной:

```
TELEGRAM_LOGIN_BOT=testminiappifbot
```

Если `.env` нет — скопируйте его из `.env.example` и задайте нужные значения.

## 3. Telegram Login Widget на localhost

В BotFather домен настроен на `www.pifagor.trade`. Виджет «Войти с Telegram» проверяет адрес страницы. На `localhost` он может не срабатывать, даже если настройки верные.

Для проверки логина локально можно использовать:

- **онлайн-версию** pifagor.trade, или
- туннель (например, ngrok) и добавить в BotFather домен вида `xxxx.ngrok.io`.

## 4. Что сделано

- Сервер читает `.env` из папки проекта
- Для localhost добавлен fallback: если `TELEGRAM_LOGIN_BOT` пустой, подставляется `testminiappifbot`
