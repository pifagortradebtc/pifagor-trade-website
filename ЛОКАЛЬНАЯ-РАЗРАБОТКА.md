# Локальная разработка — вход через Telegram

Если на localhost показывается ошибка «Настройте TELEGRAM_LOGIN_BOT», проверьте:

## 1. Запуск проекта

Запускайте **именно pifagor-trade-website** (а не pifagor-trade-redesign):

```bash
cd pifagor-trade-website
npm start
```

Откройте сайт в браузере по адресу **http://localhost:3000** (или порт из вывода терминала).

## 2. Файл .env

В папке `pifagor-trade-website` должен быть файл `.env` с переменной:

```
TELEGRAM_LOGIN_BOT=testminiappifbot
```

Если `.env` нет — скопируйте его из `.env.example` и задайте нужные значения.

## 3. Ошибка «Bot domain invalid» на localhost

Виджет загружается, но при нажатии Telegram показывает **«Bot domain invalid»**. Причина: в BotFather указан домен `www.pifagor.trade`, а локально вы открываете `localhost` или `127.0.0.1` — Telegram их не считает разрешёнными.

### Варианты

**Вариант A (рекомендуется):** проверять вход только на **pifagor.trade**. Локально тестировать остальные части без логина.

**Вариант B — ngrok для локального логина:**
1. Установите [ngrok](https://ngrok.com): `npm install -g ngrok`
2. Запустите сервер: `npm start` (порт 3000)
3. В другом терминале: `ngrok http 3000`
4. Ngrok выдаст URL вида `https://abc123.ngrok-free.app`
5. В @BotFather: `/setdomain` → выберите бота → введите домен без `https://` (например, `abc123.ngrok-free.app`)
6. Откройте сайт по ngrok-ссылке — вход должен работать  
7. После тестов верните в BotFather домен `www.pifagor.trade`

**Вариант C:** использовать `127.0.0.1` на порту 80 и указать этот домен в BotFather (тогда временно перестанет работать продакшен).

## 4. Что сделано

- Сервер читает `.env` из папки проекта
- Для localhost добавлен fallback: если `TELEGRAM_LOGIN_BOT` пустой, подставляется `testminiappifbot`
